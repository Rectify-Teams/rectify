We're excited to announce **Rectify 2.4.4** — the latest release, continuing to build on the unified 2.x line with stability improvements and refinements across all packages.

<!-- truncate -->

## What's new

### Auto-bailout for function components

Function components now skip re-renders automatically when props are shallow-equal — no need to wrap every component in `memo()`. Use `memo` only when you need a custom comparator.

### SVG support

SVG elements are now created with `createElementNS` so they render correctly in the browser. camelCase attributes (e.g. `strokeWidth`) are automatically converted to their kebab-case equivalents.

### `@rectify-dev/router` — initial public release

Rectify now ships a full client-side router:

| Feature | Details |
|---|---|
| `BrowserRouter` | HTML5 History API (`pushState` / `popstate`), optional `basename` |
| `HashRouter` | Hash-based routing for static hosting |
| `Routes` / `Route` | First-match rendering, nested layouts, `Outlet` |
| URL params | `:id`, wildcard `*`, index routes |
| `Link` / `NavLink` | Client-side navigation; `NavLink` adds `activeClassName` |
| `Navigate` | Declarative redirect-on-render |
| `useNavigate` | Programmatic push / replace / delta |
| `useLocation` | Current `pathname`, `search`, `hash`, `state`, `key` |
| `useParams` | Typed URL params |
| `useMatch` | Match any pattern against the current pathname |
| `useSearchParams` | `[URLSearchParams, setter]` with push navigation |
| `useHref` | Resolve a path to a full href string |

### Documentation site

A full Docusaurus documentation site is now live — covering the core API, hooks, router, and this blog.
